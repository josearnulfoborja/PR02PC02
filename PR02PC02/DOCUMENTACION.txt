
DOCUMENTACIÓN DEL PROYECTO PR02PC02
Sistema de Reservas de Hotel

ESTRUCTURA DEL PROYECTO:
-----------------------

1. FRONTEND:
   - Index.jsp: Página principal del sistema con navegación
   - registro.jsp: Formulario de registro de usuarios
   - app.js: JavaScript para envío de datos con Fetch API
   - estilos.css: Estilos CSS para la interfaz

2. BACKEND (Java):
   - JakartaRestConfiguration.java: Configuración REST (@ApplicationPath("resources"))
   - SrvUsuarios.java: Endpoint REST para registro de usuarios
   - JakartaEE8Resource.java: Endpoint de prueba (ping)
   - conexion.java: Clase para conexión a base de datos MySQL

3. BASE DE DATOS:
   - MySQL (localhost:3306/plataforma)
   - Credenciales: root/root


FLUJO DE REGISTRO:
-----------------
1. Usuario accede a registro.jsp
2. Completa formulario (nombre, apellido, correo, teléfono, contraseña)
3. JavaScript captura submit y envía datos JSON al backend
4. Backend procesa datos y responde con confirmación
5. Frontend muestra mensaje de éxito/error

PROSIBLES ERRORES:
---------------
- Error 404: Verificar que el servidor esté ejecutándose y desplegado
- Problemas de CORS: Asegurar headers correctos en Fetch API
- Error BD: Verificar credenciales y conexión MySQL
- JavaScript no funciona: Verificar referencia a app.js en registro.jsp
- Index: Verifica que el nombre del index inicie en minuscula, index.jsp

ERROR COMÚN DE NAVEGACIÓN:
-------------------------
Si al ejecutar comandos de terminal obtienes el error:

cd : No existe ningún objeto en la ruta de acceso especificada, C:\src\PR02PC02\PR02PC02\....
En línea: 1 Carácter: 1
+ cd ...
+ ~~~~~~
    + CategoryInfo          : InvalidArgument: (:) [Set-Location], PSArgumentException
    + FullyQualifiedErrorId : Argument,Microsoft.PowerShell.Commands.SetLocationCommand

SOLUCIÓN:
- Primero navega a la carpeta principal del proyecto:
  cd C:\src\PR02PC02
- Luego entra a la subcarpeta del proyecto:
  cd PR02PC02
- O navega directamente con la ruta completa:
  cd "C:\src\PR02PC02\PR02PC02"

NOTA: Este error ocurre porque el proyecto tiene una estructura de carpetas anidada
donde hay dos niveles de PR02PC02 (C:\src\PR02PC02\PR02PC02\)
